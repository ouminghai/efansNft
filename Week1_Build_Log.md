# Week 1 Build Log

## 1. Week 1 Build Log 初稿

本周我完成了一个基于 Monad Testnet 的 Web3 粉丝互动 DApp 初版，项目核心是用链上互动驱动粉丝身份成长。整体上，我先完成了双合约架构设计与部署，包括 `FanBadgeNFT` 合约和 `PointSystem` 合约。其中 NFT 合约负责粉丝徽章的铸造、等级展示和动态 metadata，积分合约负责签到、点赞、投票和升级等互动逻辑。部署完成后，我将两个合约地址接入前端，使前端能够直接与链上状态交互。

前端部分使用 React、wagmi、RainbowKit 和 viem 构建，重点实现了钱包连接、NFT Badge 展示、积分读取、签到/点赞/投票/升级等交互入口，并补充了交易反馈、活动记录展示和基础状态提示。过程中我也处理了若干实际问题，例如合约地址回填、wagmi 写合约参数类型问题、部分按钮点击后缺少反馈的问题，以及前端部署到 Vercel 时的依赖冲突问题。最终项目已经可以在测试网环境下完成基础钱包连接、链上交互和线上访问。

## 2. 你选择 Tech 的初步理由

我当前选择的 Tech 路线是 Solidity + Hardhat + React + wagmi/RainbowKit + Vercel，这个组合的主要原因是它比较适合快速搭建一个完整的 Web3 DApp 原型。Solidity 和 Hardhat 适合做智能合约开发、测试和部署；React 适合快速搭建交互界面；wagmi 和 RainbowKit 可以较低成本地接入钱包连接与合约读写；Vercel 则适合快速完成前端部署和分享。

另外，这套技术栈和 Monad 的高频链上交互场景也比较匹配。因为这个项目希望体现“互动 -> 积分 -> NFT 成长”的链上闭环，所以前端、钱包、合约和部署链路都需要尽量顺畅，而这套方案能让我在较短时间内完成从本地开发到测试网上线的完整流程。

## 3. 本周最重要的 1–3 个学习收获

1. 我更清楚地理解了 DApp 的完整链路，不只是写一个前端页面，而是需要把智能合约、钱包连接、链上交易反馈、前端状态展示和部署流程一起打通。
2. 我学会了双合约之间的协作方式，比如先部署 NFT 合约，再将其地址传入积分合约，并反向设置 `pointSystem` 权限，让两个合约形成完整业务闭环。
3. 我对前端接链的实际问题有了更深理解，例如合约地址配置、wagmi 合约读写的参数要求、交易确认后的 UI 更新，以及部署到 Vercel 时 Node 依赖冲突的排查方式。

## 4. 你希望助教或同伴帮助的问题

1. 我希望得到关于“如何把当前 Demo 进一步做得更像完整产品”的建议，尤其是链上活动记录、任务系统和用户成长体系该如何设计得更合理。
2. 我想请教关于智能合约测试与安全性的最佳实践，例如我当前的积分与升级逻辑还比较基础，后续该如何补测试、做权限控制和避免常见漏洞。
3. 我也希望获得一些关于 Web3 前端体验优化的建议，比如交易状态反馈、错误提示、移动端适配，以及如何更自然地向非技术用户解释链上交互过程。
