# 活动执行方案：用 AI 辅助完成 Builder 学习记录

## 活动基本信息

活动标题：用 AI 辅助完成 Builder 学习记录：从零散学习到可展示 Proof of Work

活动形式：线上 Space / AMA / 小型语音分享

活动规模：30-100 人

活动时长：60 分钟

目标人群：Web3 / AI / Builder 新人，尤其是已经开始学习但不知道如何沉淀 Proof of Work 的参与者。

核心目标：帮助新人理解如何把 AI 辅助学习、项目尝试和角色探索，整理成可展示的学习记录、README、活动复盘或 GitHub 文档。

## 时间排期

建议采用活动前 7 天启动准备的节奏。

| 时间 | 任务 | 产出 |
|---|---|---|
| T-7 天 | 确认活动主题、目标听众和核心问题 | 活动主题文档 |
| T-6 天 | 确认主持人、嘉宾和协作人员 | 人员名单与分工 |
| T-5 天 | 完成 Run of Show 和暖场案例设计 | 活动流程稿 |
| T-4 天 | 准备宣传文案、海报、报名链接或预约链接 | 宣传物料 |
| T-3 天 | 发布第一轮活动预告 | 社群 / Twitter / Discord 预告 |
| T-2 天 | 和嘉宾对齐讨论问题、案例和时间控制 | 嘉宾 briefing 文档 |
| T-1 天 | 发布第二轮提醒，测试 Space 链接和设备 | 提醒文案、设备检查表 |
| T 日 | 活动执行、互动收集、记录问题 | 活动记录和互动反馈 |
| T+1 天 | 发布活动复盘和 CTA，鼓励参与者提交学习记录 | 活动复盘、PoW 收集 |

## 宣传计划

宣传重点不是制造大流量，而是让目标新人知道“这场活动能解决我现在的具体困惑”。

### 第一轮预告：T-3 天

目标：让新人知道活动主题，并判断自己是否适合参加。

宣传重点：

- 你是否学了很多 Web3 / AI 概念，但不知道怎么展示？
- 你是否用了 AI 工具，但没有形成自己的学习记录？
- 你是否想加入团队，却说不清自己能提供什么 Proof of Work？

文案方向：

```text
学了很多，不等于别人能看见你的成长。

这场小型 Space 会聊：新人如何用 AI 辅助完成 Builder 学习记录，把零散学习变成可展示的 Proof of Work。

适合：Web3 / AI 新人、想参加组队但还没有作品的人、想整理 GitHub / README / 学习记录的人。
```

### 第二轮提醒：T-1 天

目标：提醒已感兴趣的人预约或准时参加。

宣传重点：

- 活动时间
- 适合人群
- 参与者能带走什么
- 互动环节需要准备什么

提醒文案方向：

```text
明天的 Space 会带一个新人案例：小林学了很多 Web3，也用了 AI，但组队时拿不出 Proof of Work。

我们会一起拆解：如果你是小林，怎么把聊天记录、学习笔记、README 和项目尝试整理成可展示输出？

建议提前准备一个你最近学习过的主题，活动中会现场写 3 句话 Proof of Work 草稿。
```

### 活动当天提醒：T 日开始前 1-2 小时

目标：提高到场率。

提醒重点：

- 活动即将开始
- 带着一个最近学习主题来参加
- 活动后鼓励发布一份学习记录

## 人员分工

| 角色 | 主要职责 |
|---|---|
| 主持人 | 控制节奏、串联暖场案例、引导嘉宾回答、推动互动和 Q&A |
| 分享嘉宾 | 结合自身经验回答核心问题，提供新人可模仿的方法 |
| 运营负责人 | 负责宣传文案、社群发布、报名或预约提醒、活动后复盘 |
| 记录员 | 记录活动重点、嘉宾金句、参与者问题和互动回答 |
| 技术 / 场控 | 检查 Space 链接、麦克风、录音、嘉宾上麦权限和备用沟通渠道 |
| 社群互动 | 在评论区收集问题、提醒参与者完成 3 句话 PoW 草稿 |

如果团队人数较少，可以合并角色：

- 主持人兼场控
- 运营负责人兼记录员
- 嘉宾兼 Q&A 回答者

## 活动当天执行流程

### 活动前 30 分钟

- 主持人、嘉宾、场控进入准备状态。
- 测试麦克风、网络和上麦权限。
- 确认活动链接、标题和简介是否正确。
- 在社群发布最后一次提醒。
- 主持人快速复习 Run of Show 和暖场案例。

### 活动前 5 分钟

- 主持人提前进入 Space。
- 场控确认嘉宾是否在线。
- 运营在社群提醒“活动即将开始”。
- 记录员准备记录文档。

### 0-8 分钟：暖场案例

主持人讲“小林”的故事：

小林学了很多 Web3，也问了 AI 很多问题，但组队时拿不出 Proof of Work。

抛出悬念：

```text
如果你是小林，你怎么把这两周的零散学习，整理成一份别人看得懂、愿意相信的 Proof of Work？
```

### 8-15 分钟：问题引入

主持人说明新人常见困境：

- 学习很多但没有记录
- AI 问答没有转化成输出
- 不知道如何表达自己的角色能力
- 没有 README、截图、链接或 GitHub 记录

### 15-30 分钟：案例拆解

嘉宾或主持人拆解 AI 能帮助新人完成的具体输出：

- 学习笔记
- README
- 开发计划
- 活动复盘
- 角色说明
- Proof of Work 草稿

重点强调：AI 可以辅助整理，但最终判断和验证要由人完成。

### 30-45 分钟：核心讨论

围绕核心问题讨论：

1. 新人为什么需要学习记录？
2. AI 可以帮助新人完成哪些具体输出？
3. 如何判断 AI 输出是否能放进自己的 Proof of Work？
4. 没有技术背景的人也能做 Builder 记录吗？
5. 一份好的新人 Proof of Work 应该包含什么？

### 45-55 分钟：互动与 Q&A

互动任务：

```text
第 1 句话：我正在学习或尝试的主题是____。
第 2 句话：我已经完成的具体输出是____。
第 3 句话：我下一步想验证或改进的是____。
```

主持人邀请 2-3 位参与者分享，并帮助他们把表达变得更清楚。

Q&A 回到暖场问题：

```text
如果你是小林，你会怎么补救？
```

引导答案：

- 整理 AI 问答为学习笔记
- 补一个 README
- 加入截图、链接、交易哈希或 commit
- 写清 AI 帮助了什么，自己判断了什么
- 用 3 句话说明团队角色和下一步计划

### 55-60 分钟：收尾 CTA

主持人总结：

```text
不要等到完整项目完成后才开始展示。清晰记录每一次学习和验证，本身就是 Builder 的 Proof of Work。
```

活动后 CTA：

请参与者在 24 小时内发布一份自己的 Builder 学习记录。

可选形式：

- Markdown 学习笔记
- GitHub README
- 长推或社区帖子
- 活动复盘
- Demo 截图说明
- 300-500 字角色说明

## 风险预案

| 可能问题 | 影响 | 应对方式 |
|---|---|---|
| 嘉宾临时无法参加 | 内容深度下降 | 主持人提前准备完整串场稿和备用案例，可改为主持人单人分享 + Q&A |
| Space 链接或平台异常 | 参与者无法进入 | 准备备用会议链接或备用 Space，并提前在社群同步 |
| 到场人数较少 | 互动不足 | 改为更小范围 AMA，主持人主动点名社群问题，鼓励文字区互动 |
| 参与者过多但问题分散 | 节奏失控 | 由社群互动负责人筛选问题，优先回答与 Proof of Work 相关的问题 |
| 嘉宾回答太发散 | 时间超时 | 主持人用“回到小林案例”收束讨论 |
| 参与者不知道如何参与互动 | 互动冷场 | 提前给出 3 句话模板，并由主持人先示范 |
| AI 话题变成工具推荐 | 偏离主题 | 主持人提醒重点是学习记录和 Proof of Work，不是比较工具 |
| 新人担心自己没有成果 | 参与压力变大 | 强调最小输出即可，哪怕是一篇笔记或一个 README 也算开始 |
| 活动后没有沉淀 | 影响传播和复用 | 记录员整理复盘，运营在 24 小时内发布总结和 CTA |

## 预期目标

### 参与目标

- 活动预约或报名人数：50-100 人
- 实际到场人数：30-60 人
- 互动参与人数：5-15 人

### 内容目标

- 让参与者理解 Proof of Work 不只等于完整项目，也包括学习记录、文档、复盘和验证过程。
- 让参与者知道如何用 AI 辅助整理输出，而不是只停留在聊天问答。
- 让参与者掌握一个简单模板：主题、输出、下一步验证。

### 活动后目标

- 至少收集 5 份参与者的 3 句话 Proof of Work 草稿。
- 至少有 3 位参与者在活动后发布学习记录或 README。
- 形成一份活动复盘，记录高频问题、有效案例和下一次活动可以改进的地方。

## 执行重点

这场活动的执行重点不是把流程做复杂，而是保证参与者能听懂、能参与、能带走一个行动。

最重要的三个抓手是：

1. 用“小林”的故事降低理解门槛。
2. 用 3 句话模板降低参与门槛。
3. 用活动后 24 小时 CTA 推动真实输出。

只要参与者能从“我学了很多”变成“我围绕一个问题完成了一个输出，并知道下一步要验证什么”，这场活动就达到了目标。
